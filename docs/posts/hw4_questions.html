MaxDiff Analysis of MSBA Class Preferences

Counts, MLE, and Bayesian estimation of the MNL model

Author

He Zou

Published

May 27, 2026

Introduction

MaxDiff, also called Best-Worst Scaling, is a survey method for measuring relative preferences. Instead of asking people to rate every item on a numeric scale, each respondent sees a small set of items and chooses the one they like most and the one they like least. In this survey, each task shows four MSBA classes from UCSD Rady, and the respondent selects the most-preferred and least-preferred class from that set.

MaxDiff is useful because people are usually better at making direct trade-offs than at calibrating a 1-to-10 rating scale. One person’s “8 out of 10” may not mean the same thing as another person’s “8 out of 10,” but choosing a best and worst option forces real differentiation among the items shown together. This helps avoid scale-use bias and makes the results easier to compare across respondents.

In this analysis, the items are the 10 core MSBA classes. I estimate class preferences in three ways. First, I use a simple counts analysis: the percent of times each class is picked best minus the percent of times it is picked worst. Second, I fit a multinomial logit model, or MNL model, using maximum likelihood. Third, I fit the same MNL model using Bayesian methods with a random-walk Metropolis-Hastings sampler. I expect the three methods to agree on the broad ranking of classes, especially at the top and bottom, but they differ in how they quantify the strength of preference and how they communicate uncertainty.

The Data

Each row in the raw data is one item shown in one respondent-task screen. A response of 1 means the item was selected as best, -1 means it was selected as worst, and 0 means the item was shown but not selected.

Before analysis, I keep only the true MaxDiff tasks. The uploaded raw file contains some follow-up screens that do not match the MaxDiff structure because they have only two rows and no worst pick. Those screens are excluded because this assignment focuses on the 10 labeled class items and the best-worst choice tasks.

Counts Analysis

The counts analysis is a simple descriptive summary of the MaxDiff responses. For each class, I calculate the percentage of times it was picked as best, the percentage of times it was picked as worst, and the difference between those two percentages:

\[ \text{score}_j = \%\text{best}_j - \%\text{worst}_j. \]

A score near 1 means that a class is almost always selected as best when shown. A score near -1 means that it is almost always selected as worst. A score near 0 means that the class is either middle-of-the-pack or potentially polarizing.

From MaxDiff Data to MNL Choices

The MaxDiff data can also be treated as multinomial logit choice data. The MNL model assigns each item \(j\) a utility coefficient \(\beta_j\). If a task shows items \(\{a, b, c, d\}\), the probability that item \(b\) is chosen as best is

\[ P(\text{best} = b) = \frac{\exp(\beta_b)} {\exp(\beta_a) + \exp(\beta_b) + \exp(\beta_c) + \exp(\beta_d)}. \]

After the best item is selected, the worst item is chosen from the remaining three items. Since the worst item is the least preferred item, the utilities are flipped:

\[ P(\text{worst} = c \mid \text{best} = b) = \frac{\exp(-\beta_c)} {\exp(-\beta_a) + \exp(-\beta_c) + \exp(-\beta_d)}. \]

Therefore, each MaxDiff task contributes two MNL-style observations: one best choice from the full set of four items, and one conditional worst choice from the remaining three items.

The model is identified only up to an additive constant. If the same constant is added to every \(\beta\), the soft-max probabilities do not change. I therefore normalize the first class to have \(\beta_1 = 0\) and estimate the other coefficients relative to that reference class.

MNL via Maximum Likelihood

For one task, the log-likelihood contribution is the log probability of the observed best choice plus the log probability of the observed conditional worst choice. Summing over all tasks,

\[ \ell(\boldsymbol{\beta}) = \sum_{\text{tasks}} \left[ \beta_{j_b^*} - \log\sum_{j \in \text{shown}}\exp(\beta_j) - \beta_{j_w^*} - \log\sum_{j \in \text{shown}\setminus \{j_b^*\}}\exp(-\beta_j) \right], \]

where \(j_b^*\) is the selected best item and \(j_w^*\) is the selected worst item.

The following code is the core MNL likelihood and optimizer code. It is shown because it is the main model-building part of the analysis. The first class is pinned to zero for identification, and the optimizer estimates the remaining nine coefficients.

def full_beta(theta):
    """theta has length J-1. The first beta is pinned to zero."""
    beta = np.zeros(J_total)
    beta[1:] = theta
    return beta


def loglik(theta):
    beta = full_beta(theta)
    best_part = beta[best_idx] - row_logsumexp(beta[shown])
    worst_part = -beta[worst_idx] - row_logsumexp(-beta[remaining])
    return float((best_part + worst_part).sum())


def neg_loglik(theta):
    return -loglik(theta)


def grad_loglik(theta):
    beta = full_beta(theta)
    grad = np.zeros(J_total)

    # Best-choice contribution: chosen item minus softmax probabilities over shown set.
    v = beta[shown]
    p_best = np.exp(v - row_logsumexp(v)[:, None])
    np.add.at(grad, best_idx, 1.0)
    np.add.at(grad, shown.ravel(), -p_best.ravel())

    # Worst-choice contribution: chosen worst item has -1;
    # remaining items get flipped-utility probabilities.
    u = -beta[remaining]
    p_worst = np.exp(u - row_logsumexp(u)[:, None])
    np.add.at(grad, worst_idx, -1.0)
    np.add.at(grad, remaining.ravel(), p_worst.ravel())

    # Drop reference coefficient.
    return grad[1:]


def grad_neg_loglik(theta):
    return -grad_loglik(theta)


mle_out = minimize(
    neg_loglik,
    x0=np.zeros(J_total - 1),
    jac=grad_neg_loglik,
    method="BFGS",
    options={"gtol": 1e-7, "maxiter": 2000},
)
if not mle_out.success:
    display(Markdown(f"**Warning:** optimizer message: `{mle_out.message}`"))

# Finite-difference Hessian of the negative log-likelihood gradient.
def finite_diff_jacobian(func, x, eps=1e-5):
    x = np.asarray(x, dtype=float)
    K = len(x)
    J = np.empty((K, K))
    for k in range(K):
        step = eps * max(1.0, abs(x[k]))
        x_plus = x.copy()
        x_minus = x.copy()
        x_plus[k] += step
        x_minus[k] -= step
        J[:, k] = (func(x_plus) - func(x_minus)) / (2 * step)
    return J


hess_nll = finite_diff_jacobian(grad_neg_loglik, mle_out.x)
hess_nll = (hess_nll + hess_nll.T) / 2
cov_mle = np.linalg.inv(hess_nll)
se_free = np.sqrt(np.diag(cov_mle))

beta_mle = full_beta(mle_out.x)
se_mle = np.r_[np.nan, se_free]
mle_share = softmax(beta_mle)

mle_tbl = item_map.copy()
mle_tbl["beta_mle"] = beta_mle
mle_tbl["se_mle"] = se_mle

mle_tbl["mle_rank"] = np.arange(1, len(mle_tbl) + 1)

In this dataset, the MLE ranking is identical to the counts ranking. This similarity makes sense because both approaches use the same best-worst information. The advantage of the MNL model is that it explicitly models the choice process: every item shown in a task contributes to the probability denominator, so the model uses not only which items were picked, but also which alternatives were available and not picked.

MNL via Bayesian Estimation

The Bayesian version uses the same likelihood but adds a prior distribution over the free \(\beta\) parameters:

\[ \pi(\boldsymbol{\beta} \mid \boldsymbol{y}) \propto L(\boldsymbol{\beta})\pi(\boldsymbol{\beta}). \]

I use a weakly informative normal prior,

\[ \boldsymbol{\beta} \sim N(\boldsymbol{0}, 10I). \]

This prior is centered at zero and has standard deviation \(\sqrt{10} \approx 3.16\), which is broad enough to be weakly informative on the utility scale.

The following code is the core Bayesian model code. It defines the posterior and runs a random-walk Metropolis-Hastings sampler. It is shown because it is the main Bayesian estimation algorithm.

prior_sd = np.sqrt(10.0)
log_prior_const = -0.5 * (J_total - 1) * np.log(2 * np.pi * prior_sd**2)


def log_posterior(theta):
    theta = np.asarray(theta)
    return loglik(theta) + log_prior_const - 0.5 * np.sum(theta**2) / prior_sd**2


def mh_sampler(theta0, n_iter, step, seed=1234):
    local_rng = np.random.default_rng(seed)
    theta = np.array(theta0, dtype=float).copy()
    K = len(theta)
    draws = np.empty((n_iter, K))
    lp = log_posterior(theta)
    accept = 0

    for t in range(n_iter):
        proposal = theta + local_rng.normal(0, step, size=K)
        lp_proposal = log_posterior(proposal)

        if np.log(local_rng.random()) < lp_proposal - lp:
            theta = proposal
            lp = lp_proposal
            accept += 1

        draws[t, :] = theta

    return draws, accept / n_iter


def tune_step(theta0, start_step=0.08, n_pilot=2000, max_rounds=6):
    step = start_step
    current = np.array(theta0, dtype=float).copy()
    history = []

    for r in range(max_rounds):
        pilot_draws, ar = mh_sampler(current, n_pilot, step, seed=9000 + r)
        current = pilot_draws[-1]
        history.append({"round": r + 1, "step": step, "accept_rate": ar})

        if 0.25 <= ar <= 0.40:
            break
        elif ar < 0.25:
            step = step / 2
        else:
            step = step * 2

    return step, current, pd.DataFrame(history)

I start the chain at the MLE to reduce warm-up time. I then tune the proposal step size with short pilot runs, targeting an acceptance rate around 25% to 40%.

Overall, the three methods agree most clearly at the extremes. The classes with the strongest positive signals stay near the top, and the classes with the strongest negative signals stay near the bottom. In this data, the Bayesian top three are MGTA 495 Marketing Analytics (Spring with Yavorsky); MGTA 453 Business Analytics (Fall with August); MGTA 455 Customer Analytics (Winter with Nijs), while the Bayesian bottom three are MGTA 444 Analytics Consulting (Winter with Peterson); MGTA 451 Marketing/Finance/Operations (Summer with Wilbur/Buti/Shin); MGTA 403 AI Math, Prog., & Analytics (Summer with Nijs). In this cleaned MaxDiff dataset, the three methods give the same ranking for all 10 classes. The MNL model adds structure relative to the counts analysis because it uses the full shown set in each task. The Bayesian model adds a convenient way to quantify uncertainty for derived quantities such as shares of preference.

Discussion

The substantive conclusion is that MSBA students do not view the core classes as interchangeable. Based on the Bayesian MNL estimates, the most preferred classes are MGTA 495 Marketing Analytics (Spring with Yavorsky); MGTA 453 Business Analytics (Fall with August); MGTA 455 Customer Analytics (Winter with Nijs). The least preferred classes are MGTA 444 Analytics Consulting (Winter with Peterson); MGTA 451 Marketing/Finance/Operations (Summer with Wilbur/Buti/Shin); MGTA 403 AI Math, Prog., & Analytics (Summer with Nijs). I would be most cautious about interpreting exact differences in the middle of the ranking because those classes have closer estimated utilities and more overlapping uncertainty intervals.

Methodologically, the three approaches answer related but different questions. The counts analysis is best as a quick dashboard metric: it is transparent, fast, and easy to communicate. The maximum-likelihood MNL model is better when I want a formal model of the choice process, point estimates, and classical standard errors. The Bayesian MNL model is most useful when I care about uncertainty in derived quantities, such as shares of preference or future simulations, because every posterior draw can be pushed through the calculation of interest.

There are also important caveats. This survey reflects a self-selected sample of MSBA students, so the results may not generalize to all students or future cohorts. Preferences may also depend on prior coursework, career goals, current standing in the program, instructor effects, and the order or context in which classes were presented. If I had more time, I would extend the model to allow respondent-level heterogeneity so that different student segments could have different class preference patterns.
